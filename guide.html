How to Submit Your Answers

△ Σ Ξ ᚠ ᚱ • Keep your scroll tidy • ᚨ ᚾ ᛊ

1) Generate your token

  • Select a question (Q1, Q2, …).
  • Enter your GitHub username and your answer.
  • Press Enter to generate the token shown under Hash:.
  • Use the COPY HASH button to copy it.

2) Create the submission file

Create a file per question with this exact path pattern:

submissions/<your-github-username>/<your-github-username>_q<N>.txt

File contents must be a single line:

<hash>:<nonce>

Example for user octocat submitting Q3:

submissions/octocat/octocat_q3.txt
# contents:
2f5a...be9e:ab12cd34

3) Commit, push, and open a PR

git add submissions/<you>/<you>_q<N>.txt
git commit -m "Submit Q<N> answer"
# ensure your fork main is up to date
git pull --rebase origin main
git push origin main

On GitHub, click Compare & pull request, title it like Q<N> answer by <you>, and submit.

4) What CI checks

  • Path check: files must match submissions/<you>/<you>_q<N>.txt.
  • Answer verification: derives sha256(sha256(username)+sha256(correct_answer)+nonce) and matches your <hash>.
  • Commit signature: fails some bot-style merges.

If a check fails, open the PR Checks tab for details and fix the path/token.

Tips

  • Username in the path must match your PR author login.
  • Keep the submission line exactly <hash>:<nonce> (no spaces).
  • Use Hint Mode and the media links (video/image) if provided.
